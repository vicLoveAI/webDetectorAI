# 网页内容 AI 摘要器

一键探测网页内容概要，流式 Markdown 输出，支持自定义大模型 API。

## 功能特性

- **一键探测** — 点击工具栏图标、右键菜单或快捷键 `Ctrl+Shift+S`（Mac: `Cmd+Shift+S`）
- **流式输出** — AI 生成内容实时逐字显示，无需等待完整响应
- **Markdown 渲染** — 结果以格式化 Markdown 展示，支持一键复制
- **多模态视觉支持** — 图片/视频页面将实际图片发送给视觉模型（GPT-4o、Claude、Gemini 等）进行视觉内容分析，不再仅依赖文本元数据
- **内容类型识别** — 自动识别文章/视频/音频/图片页面，采用不同的提取和摘要策略
- **Token 预估** — 调用前预估 Token 消耗，高用量时弹窗确认，避免意外费用
- **多语言输出** — 跟随网页语言或指定输出语言（中文、英语、日语、韩语、法语、德语）
- **自定义 API** — 支持 OpenAI 及所有兼容接口（DeepSeek、Qwen、LiteLLM 等）
- **模型查询** — 一键获取 API 可用模型列表
- **连接测试** — 测试 API 配置是否可用

## 安装方法

1. 打开 Chrome 浏览器，进入 `chrome://extensions/`
2. 开启右上角 **"开发者模式"**
3. 点击 **"加载已解压的扩展程序"**
4. 选择本扩展所在的文件夹（包含 `manifest.json` 的目录）
5. 扩展图标将出现在浏览器工具栏中

## 配置步骤

### 1. 获取 API Key

#### OpenAI 官方
1. 访问 [https://platform.openai.com/api-keys](https://platform.openai.com/api-keys)
2. 注册/登录，创建 API Key
3. 在扩展设置中填入：
   - API 基础地址：`https://api.openai.com/v1`
   - API Key：`sk-...`
   - 点击「📡 查询模型」获取可用模型列表

#### DeepSeek
1. 访问 [https://platform.deepseek.com/api_keys](https://platform.deepseek.com/api_keys)
2. 获取 API Key
3. 在扩展设置中填入：
   - API 基础地址：`https://api.deepseek.com/v1`
   - 模型名称：`deepseek-chat`（或点击「查询模型」查看可用列表）
   - **注意**：DeepSeek 模型不支持视觉/图片分析

#### 其他兼容接口
任何兼容 OpenAI Chat Completions API 的服务均可使用。如需图片视觉分析，请使用支持多模态输入的模型（GPT-4o、GPT-4.1、Claude 3/4、Gemini 1.5/2.0 等）。

### 2. 测试连接（推荐）

1. 点击 **「🔌 测试连接」** 按钮验证 API 是否可用
2. 成功会显示「✅ 连接成功」及可用模型数量
3. 失败会显示具体错误原因

### 3. 查询可用模型

1. 输入 API 地址和 Key
2. 点击 **「📡 查询模型」** 按钮
3. 从下拉列表中点击选择模型名称

## 使用说明

1. 打开任意网页（博客文章、新闻、视频页面、图片集等）
2. **方式一**：点击工具栏扩展图标 → 点击 **「🔍 一键探测」**
3. **方式二**：在网页上右键 → 选择 **「🔍 一键探测网页概要」**
4. **方式三**：使用快捷键 `Ctrl+Shift+S`（Mac: `Cmd+Shift+S`）
5. 摘要内容将**实时流式显示**，无需等待完整生成
6. 点击 **「📋 复制 Markdown」** 可复制原始文本
7. 生成过程中可点击 **「✕ 取消」** 中断生成

## 内容提取策略

| 页面类型 | 提取内容 | 摘要格式 |
|---------|---------|---------|
| 文章/博客 | 标题、描述、正文（最多 8000 字） | 标题 → 摘要 → 要点列表 → 结论 |
| 视频页面 | 标题、作者/频道、时长、描述、**缩略图/关键帧用于视觉分析** | 视频信息 → 画面分析 → 内容概要 → 关键看点 |
| 音频/播客 | 标题、作者、描述、**转录文本/歌词/节目笔记** | 音频信息 → 内容概要 → 关键内容 |
| 图片集 | **图片 URL（最多 5 张）用于视觉分析**、alt 文本、页面上下文 | 图片集概要 → 逐图详细分析 |

**视觉模式**自动启用的条件：
- 页面类型为图片、视频或包含视觉元素的音频页面
- 配置的模型支持多模态输入（如 GPT-4o、Claude、Gemini）
- 如果模型不支持视觉，扩展会自动回退到纯文本模式

## Token 用量预估

- 调用 LLM API 前自动预估 Token 消耗
- 正常用量（< 3000 tokens）：直接调用
- 较高用量（3000+ tokens）：弹窗提示
- 超出模型限制 90%+：弹窗警告
- 视觉模式下图片 Token 一并计入预估（每张图片约 85 tokens）
- 用户可在弹窗中确认继续或取消

## 隐私与安全

- 本扩展**不收集任何用户数据**
- API Key 使用 Chrome `storage.sync` 加密本地存储
- API Key 输入框为密码类型，支持 👁️ 显示/隐藏切换
- 所有 API 请求通过 background service worker 代理，使用 HTTPS 加密
- 网页文本内容仅发送到用户自行配置的第三方 API 端点
- 无广告、无追踪、无遥测

## 技术栈

- Manifest V3
- 原生 HTML/CSS/JS（无大型框架依赖）
- 内置轻量 Markdown 渲染器（零依赖）
- OpenAI Chat Completions API 流式传输（SSE）
- Background Service Worker 代理所有 API 请求
- 无需 `host_permissions`（符合 Chrome 应用商店要求）
