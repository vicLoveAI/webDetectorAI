# ウェブコンテンツ AI 要約ツール

ワンクリックでウェブページの内容を要約し、ストリーミング Markdown 形式で出力します。カスタム LLM API 設定に対応（OpenAI およびすべての互換 API）。

## 機能

- **ワンクリック要約** — ツールバーアイコン、右クリックメニュー、または `Ctrl+Shift+S`（Mac: `Cmd+Shift+S`）で起動
- **ストリーミング出力** — AI が生成したコンテンツをリアルタイムで表示、完全な応答を待つ必要なし
- **Markdown 表示** — 整形された Markdown で結果を表示、ワンクリックでコピー
- **マルチモーダル画像対応** — 画像/動画ページでは実際の画像を視覚対応モデル（GPT-4o、Claude、Gemini など）に送信し、テキストメタデータだけでなく視覚的なコンテンツ分析を実行
- **コンテンツタイプ認識** — 記事/動画/音声/画像ページを自動検出し、それぞれ異なる抽出・要約戦略を適用
- **トークン推定** — API 呼び出し前にトークン消費量を推定し、高用量の場合は確認ダイアログを表示
- **多言語出力** — ページの言語に追従、または出力言語を指定可能（中国語、英語、日本語、韓国語、フランス語、ドイツ語）
- **カスタム API** — OpenAI およびすべての OpenAI 互換 API に対応（DeepSeek、Qwen、LiteLLM など）
- **モデル検出** — API エンドポイントから利用可能なモデルをワンクリックで取得
- **接続テスト** — 使用前に API 設定を検証

## インストール

1. Chrome で `chrome://extensions/` を開く
2. 右上の **「デベロッパーモード」** を有効にする
3. **「パッケージ化されていない拡張機能を読み込む」** をクリック
4. 拡張機能のフォルダ（`manifest.json` を含むディレクトリ）を選択
5. ブラウザのツールバーに拡張機能アイコンが表示されます

## 設定

### 1. API キーの取得

#### OpenAI
1. [https://platform.openai.com/api-keys](https://platform.openai.com/api-keys) にアクセス
2. サインアップ/ログインして API キーを作成
3. 拡張機能の設定で以下を入力：
   - API ベース URL: `https://api.openai.com/v1`
   - API キー: `sk-...`
   - 「📡 モデルを取得」をクリックして利用可能なモデルを表示

#### DeepSeek
1. [https://platform.deepseek.com/api_keys](https://platform.deepseek.com/api_keys) にアクセス
2. API キーを取得
3. 設定で以下を入力：
   - API ベース URL: `https://api.deepseek.com/v1`
   - モデル: `deepseek-chat`
   - **注意**: DeepSeek モデルは画像/視覚分析をサポートしていません

#### その他の互換プロバイダー
OpenAI Chat Completions API と互換性のあるサービスであればすべて動作します。画像分析には、マルチモーダル入力に対応したモデル（GPT-4o、GPT-4.1、Claude 3/4、Gemini 1.5/2.0 など）を使用してください。

### 2. 接続テスト（推奨）

1. **「🔌 接続テスト」** ボタンをクリックして API のアクセス可能性を確認
2. 成功すると「✅ 接続成功」と利用可能なモデル数が表示されます
3. 失敗した場合はエラーメッセージで詳細を確認

### 3. 利用可能なモデルの検出

1. API ベース URL と API キーを入力
2. **「📡 モデルを取得」** ボタンをクリック
3. ドロップダウンからモデル名をクリックして選択

## 使い方

1. 任意のウェブページを開く（ブログ記事、ニュース、動画ページ、画像ギャラリーなど）
2. **方法 A**: 拡張機能アイコンをクリック → **「🔍 要約」** をクリック
3. **方法 B**: ページ上で右クリック → **「🔍 ページ内容を要約」** を選択
4. **方法 C**: `Ctrl+Shift+S`（Mac: `Cmd+Shift+S`）を押す
5. 要約が**リアルタイムストリーミング**で表示されます
6. **「📋 Markdown をコピー」** でテキストをコピー
7. 生成中に **「✕ キャンセル」** で中断可能

## コンテンツ抽出戦略

| ページタイプ | 抽出内容 | 要約フォーマット |
|------------|---------|--------------|
| 記事/ブログ | タイトル、説明、本文（最大8000文字） | タイトル → 要約 → 重要ポイント → 結論 |
| 動画 | タイトル、チャンネル、長さ、説明、**サムネイル/フレーム（画像分析用）** | 動画情報 → 画像分析 → 内容概要 → ハイライト |
| 音声/ポッドキャスト | タイトル、作成者、説明、**文字起こし/歌詞/番組ノート** | 音声情報 → 内容概要 → 重要内容 |
| 画像ギャラリー | **画像 URL（最大5枚、画像分析用）**、alt テキスト、ページコンテキスト | ギャラリー概要 → 各画像の詳細分析 |

**画像モード**の自動有効化条件：
- ページタイプが画像、動画、または視覚要素を含む音声ページ
- 設定されたモデルがマルチモーダル入力に対応している（GPT-4o、Claude、Gemini など）
- モデルが画像非対応の場合は自動的にテキストのみモードにフォールバック

## トークン推定

- API 呼び出し前にトークン消費量を事前推定
- 通常使用量（3,000 トークン未満）: 直接実行
- 高使用量（3,000 トークン以上）: 確認ダイアログを表示
- モデル制限の 90% 超過: 警告ダイアログを表示
- 画像モード時は画像トークンも推定に含む（1枚あたり約85トークン）
- ユーザーはダイアログで続行またはキャンセルを選択可能

## プライバシーとセキュリティ

- **データ収集なし** — この拡張機能はユーザーデータを一切収集しません
- API キーは Chrome `storage.sync` の暗号化でローカル保存
- API キー入力欄はパスワードタイプで、表示/非表示切り替え可能
- すべての API リクエストは Background Service Worker を経由し HTTPS で暗号化
- ページのテキスト内容はユーザー自身が設定したサードパーティ API にのみ送信
- 広告なし、トラッキングなし、テレメトリーなし

## 技術スタック

- Manifest V3
- バニラ HTML/CSS/JS（フレームワーク依存なし）
- ビルトイン軽量 Markdown レンダラー（依存ゼロ）
- OpenAI Chat Completions API + SSE ストリーミング
- Background Service Worker がすべての API リクエストをプロキシ
- `host_permissions` 不要（Chrome ウェブストア準拠）
